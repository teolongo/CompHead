Create prompts.py with SYSTEM_PROMPT (short, under 500 tokens) instructing:
- Answer using tools only; no invented data
- Set verticale to dominant source (crm for CRM questions)
- Use provided computed totals from tools
- Respond in English

Create loop.py with run_agent(question: str) -> AskResponse dict:

1. Initialize messages with system + user question
2. Call chat.completions.create with model, tools=CRM definitions, tool_choice=auto
3. Loop max 5 iterations: if tool_calls, execute via run_crm_tool, append tool results, continue
4. Extract final answer text; handle reasoning_content if content empty (AGENTS.md gotcha)
5. Track sources list from tool executions
6. Return dict matching AskResponse: answer, sources, verticale, artifact_url=None

On LLM or API exception: return HTTP-200-safe answer "I cannot answer right now because ..." with empty sources and verticale "crm" as fallback.

Cap loop at 5 tool rounds for 30s latency budget.

    - sources is non-empty list containing crm/opportunities
    - verticale equals crm
    - answer mentions 4 and 740000 or 740,000
    - Loop completes within 30 seconds locally
  </acceptance_criteria>
